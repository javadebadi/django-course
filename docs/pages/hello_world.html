Start Django Project

Now that we have install django in our virtual environement, it's time to start a django project. We are in first_django_project directory. We want to start a project in this directory. Let's start a project with the name of my_site.

(.venv)
javad@DESKTOP-2GOSU3N MINGW64 ~/first_django_project
$ django-admin startproject my_site .

Note the . at the end of the command. This command tells the django to start a new project in current directory and call it my_site. After this commands you should have the following directories and files:

Templates

Django is a web framework. Thus, it needs a mechanism to generate HTML pages. HTML pages which django can generate have two parts:

  1. Static part: parts in HTML pages which are fixed.
  2. Dynamic part: parts in HTML pages which will be filled by data (Usually data which is pulled from database).

The mechanism which does this is called Template. A template has a special syntax which determines how dyanmic data must be inserted to HTML.

Rendering

In Django, we need to create an HTML string using a context data. Rendering is a action which fills the template with context data and returning the results as string.
